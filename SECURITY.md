# Security Policy

## Reporting a vulnerability

Please **do not** open a public issue for security problems.

Report privately to the maintainers. Include:

- what the issue is and where (a URL, endpoint, or file),
- steps to reproduce, ideally a small proof of concept,
- the impact you believe it has.

You'll get an acknowledgement within **72 hours**. Please give a reasonable window to patch before any public disclosure.

## Scope

**In scope**

- the Bento Cards web app and its API routes,
- the source in this repository.

**Out of scope**

- issues that require a compromised Bento account or a leaked API key,
- rate-limiting / denial-of-service on public endpoints (they're best-effort by design),
- vulnerabilities in third-party dependencies with no working exploit against this app (please report those upstream),
- automated scanner output with no demonstrated impact.

## Supported versions

Bento Cards is a continuously-deployed web app — only the currently deployed version is supported.

Thanks for helping keep scouts safe.
